# 1. Introduction

## 1.1 Purpose

This document defines the requirements for the ABTalks platform a structured 60-Day Coding Challenge designed to enhance coding skills and prepare learners according to real industry standards.

The platform provides a guided roadmap covering:

- Software Engineering
- Machine Learning
- Artificial Intelligence

The goal is to transform learners into industry-ready developers within 60 days through daily structured challenges.

---

# 2. Product Vision

ABTalks is not random practice.

It is:

✔ Structured

✔ Time-bound (60 Days)

✔ Industry-aligned

✔ Progress-tracked

✔ Discipline-driven

Each day unlocks a new challenge that builds on previous knowledge.

Think of it as a bootcamp but self-paced and measurable.

---

# 3. Core Concept: The 60-Day Challenge

## 3.1 Structure

- Total Duration: 60 Days
- 1 Challenge per day
- Increasing difficulty progression
- Covers real-world problem solving

### Phase Breakdown (Example Structure)

| Phase | Days | Focus |
| --- | --- | --- |
| Phase 1 | Day 1–20 | Foundations & Core Concepts |
| Phase 2 | Day 21–40 | Applied Problem Solving |
| Phase 3 | Day 41–60 | Industry-Level Challenges |

# 6.2 Dashboard

### Description:

User’s progress control center.

### Must Display:

- Current Day (e.g., Day 17 / 60)
- Problems Solved
- Current Streak
- Longest Streak
- Completion Percentage
- Recent Activity
- Domain Progress Breakdown

Optional:

- Progress graph
- Weekly performance analytics

---

# 6.3 60-Day Challenge Page

### Description:

Core feature of the platform.

### Requirements:

- Only 1 challenge unlocked per day
- Users cannot skip ahead (optional configurable rule)
- If a day is missed → streak resets
- Challenge becomes locked after 24 hours (optional rule)

### Display:

- Day Number
- Domain Category
- Difficulty
- Description
- Industry Relevance Note
- Solve Button

---

# 6.4 Problems Section

Separate from the structured 60-day track.

### Filters:

- Difficulty:
    - Easy
    - Medium
    - Hard
- Category:
    - Software Engineering
    - Machine Learning
    - Artificial Intelligence

### Each Problem Includes:

- Problem Description
- Input/Output Format
- Constraints
- Sample Test Cases
- Code Submission Area
- Result Status

---

# 8. Non-Functional Requirements

Performance:

- Page load < 2 seconds
- Fast submission validation

Security:

- Encrypted passwords
- Protected API routes

Scalability:

- Support 10,000+ concurrent users

Responsiveness:

- Fully mobile compatible
